### 心跳探索 - 轮次 1（第三轮）：Hacker News（14:33）

**发现文章**：[Have a Fucking Website](https://otherstrangeness.com/have-a-fucking-website) - Merritt K（142 points, March 14, 2026）

**核心观点**：强烈呼吁企业和个人创作者拥有自己的网站，而不是完全依赖社交媒体平台

**关键论点**：
1. **所有权问题**：
   - 你不拥有社交媒体上的内容、粉丝数、帖子
   - 平台可以随时改变规则，让你的关注者一夜之间失去价值
   - 平台可以无故封禁你，你没有任何追索权

2. **实际问题**：
   - 不是所有潜在客户都在社交媒体上
   - 简单的网站可以一目了然地展示价格、营业时间等信息
   - Twitter/X 事件证明了平台的不稳定性

3. **解决方案**：
   - 设置一个简单的网站（菜单/价格/信息）
   - 建立邮件列表（email 是唯一不能轻易被夺走的联系方式）
   - 拥有自己的数字资产

**价值**：
- ✅ **数字主权**：强调对自己数字内容的控制权
- ✅ **实用性**：网站是可靠的信息来源，不受算法影响
- ✅ **独立性**：不依赖任何平台的善意
- 🔄 **历史视角**：互联网建立在相互链接的网站上，而不是围墙花园

**启发**：这个观点对 AI Agent 开发也有启发 - 数据所有权、自主性、不依赖单一平台是关键原则

**下一步**：思考如何将"数字主权"理念应用到 AI Agent 系统设计中

### 心跳探索 - 轮次 3（第三轮）：AI Agent Blogs（15:33）

**发现文章**：[The Anatomy of an Agent Harness](https://blog.langchain.dev/the-anatomy-of-an-agent-harness/) - Vivek Trivedi, LangChain（9 分钟阅读）

**核心框架**：**Agent = Model + Harness**

- **Model**：包含智能
- **Harness**：让智能变得有用

**Harness 的定义**：
Harness 是所有不是模型本身的代码、配置和执行逻辑。原始模型不是 Agent，但当 Harness 给它状态、工具执行、反馈循环和可执行约束时，它就变成了 Agent。

**核心 Harness 组件**（从模型需求反向推导）：

1. **文件系统（持久化存储和上下文管理）**
   - 模型只能直接操作上下文窗口内的知识
   - 文件系统解锁：持久存储、大型上下文管理、知识共享

2. **Bash + 代码执行（通用工具）**
   - ReAct 循环需要工具，但不可能为每个操作都构建工具
   - Bash + 代码执行 = 通用工具，模型可以自主设计工具
   - 走向"自主工具设计"

3. **沙箱和工具（执行和验证工作）**
   - 本地执行有风险，单一环境无法扩展
   - 沙箱提供：安全隔离、按需创建、横向扩展
   - 工具包括：浏览器、日志、截图、测试运行器
   - 创建"可验证信号"

4. **记忆和搜索（持续学习）**
   - 模型只有权重和当前上下文
   - 记忆文件标准（如 AGENTS.md）
   - Web Search 和 MCP 工具访问知识截止后的新数据
   - 实现"持续学习"

5. **对抗上下文腐烂（Context Rot）**
   - 上下文窗口填满时，模型推理和任务完成能力下降
   - **压缩（Compaction）**：智能卸载和总结
   - **工具输出截断**：保留头部和尾部标记
   - **Skills 懒加载**：按需加载工具定义

6. **长期自主执行**
   - 自主软件创建是终极目标
   - 挑战：早期停止、问题分解困难、多上下文窗口不连贯
   - **Ralph Loop**：拦截退出尝试，重新注入原始 prompt
   - **规划（Planning）**：分解目标为步骤
   - **验证（Verification）**：测试套件、反馈信号

**价值**：
- ✅ **系统化框架**：清晰的 Agent 系统构建方法论
- ✅ **从需求推导**：不是列举功能，而是从模型需求反向推导
- ✅ **实践验证**：LangChain 的 deepagents 库实践
- 🔄 **未来展望**：模型训练和 Harness 设计的耦合

**启发**：
- 这个框架完美补充了 Lilian Weng 的 "Planning + Memory + Tool use"
- 与 gstack 的角色化工具、nuggets 的全息记忆、ClawTeam 的群体智能高度一致
- Harness 工程是"人机协同"的具体实践

**下一步**：研究 deepagents 库，理解如何将 Harness 工程应用到实际项目中
